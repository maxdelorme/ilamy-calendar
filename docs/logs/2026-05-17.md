# Development Log - 2026-05-17

## Changes

- **[callbacks]**: Added `dateEventType` prop (`'Dayjs' | 'Date' | 'string'`, default `'Dayjs'`) to control `start`/`end` types in `onEventAdd`, `onEventUpdate`, and `onEventDelete`. Internal state stays Dayjs; conversion happens in `useCalendarEngine` via `formatEventForCallback`.
- **[types]**: `IlamyCalendarProps` is a discriminated union on `dateEventType` — `onEvent*` handlers get `CalendarEventWithDateType<'Dayjs' | 'Date' | 'string'>` matching the prop (no `IlamyCalendarProps<T>`). `IlamyCalendarEventMutationHandler` union used internally for provider/engine wiring.
- **[header]**: Added `hideExportButton` prop (default `false`) to hide iCalendar export controls in the default header on desktop and mobile.
