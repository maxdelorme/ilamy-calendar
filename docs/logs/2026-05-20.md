# Development Log - 2026-05-20

## Changes

- **[recurrence / delete callbacks]**: `deleteRecurringEvent` returns `DeleteRecurringEventResult` (`events`, `updatedRecurringEvent?`, `deletedEvents?`). `useCalendarEngine` no longer calls `onEventDelete` for every scoped delete — it applies the handler result first, then notifies:
  - **`this`** (generated instance): `onEventUpdate` on base with new EXDATE
  - **`following`**: `onEventUpdate` on base with UNTIL
  - **`all`**: `onEventDelete` **once** with the stored **base** row (`id` = base id, `uid` = `getEventParentUID(baseEvent)`), even when the user clicked a synthetic instance id; overrides are removed from `events` but not notified separately
  - **`this`** (detached override): `onEventDelete` for the override only; base `exdates` unchanged, no `onEventUpdate`
- **[recurrence / scope all / state]**: Scope `"all"` removes all rows where `getEventParentUID(event) !== seriesUid` (single membership check; no redundant `event.uid` compare).
- **[recurrence / scope all / uid on update]**: Scope `"all"` update keeps `uid: getEventParentUID(baseEvent)` so `onEventUpdate` is not cleared when `updates` spread `uid: undefined`.
- **[recurrence / edit callbacks]** (continued from `1002525` / `e28574a`): `updateRecurringEvent` returns `RecurringUpdateResult` (`events`, `updated`, `added`); engine calls `onEventUpdate` / `onEventAdd` with persisted rows, not synthetic instance ids.
- **[recurrence / scope this / edit]**: New override `{baseId}_modified_*`; re-edit updates override in place by `targetEvent.id`.
- **[docs]**: Documented scope `"all"` → single `onEventDelete` on base (`rrule` + `uid`); consumer deletes whole series. Updated `AGENTS.md`, `docs/hooks-and-context.md`, `docs/types-and-interfaces.md`, `docs/testing-guide.md`, copilot instructions, `2026-05-19.md`.
- **[tests / stability + type-check]**: Fixed broken test run from 2026-05-20 CI by stabilizing the recurring UID test input (`events` array identity), adding safe optional callback invocations in recurring delete flow, and tightening provider callback mock typing so TypeScript infers `mock.calls` correctly.
